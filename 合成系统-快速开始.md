# 合成系统快速开始指南

## 🚀 快速开始 (5分钟)

### 步骤1: 创建FairyGUI界面资源

#### 1.1 创建A_Compound包
1. 打开FairyGUI编辑器
2. 新建包 → 命名为 `A_Compound`
3. 创建主界面组件 `compoundMain`

#### 1.2 界面元素创建

**左侧菜单区域:**
```
- list_group1 (GList): 一级分组列表
  ├─ item renderer: btn_group1 (GButton)
  │   ├─ title (GTextField): 分组名称
  │   └─ sel (GImage): 选中背景
  
- list_group2 (GList): 二级分组列表
  ├─ item renderer: btn_group2 (GButton)
  │   ├─ title (GTextField): 子分组名称
  │   └─ sel (GImage): 选中背景
```

**右侧内容区域:**
```
- 材料消耗标题 (GTextField): "材料消耗"
- cost_items_container (GComponent): 材料容器
  ├─ CostItem (GComponent): 材料项模板
  │   ├─ icon (GImage): 道具图标
  │   ├─ txt_need (GTextField): 需要数量
  │   └─ txt_have (GTextField): 拥有数量

- 合成获得标题 (GTextField): "合成获得"
- target_icon (GImage): 目标道具图标
- target_count (GTextField): 目标数量 "×1"

- currency_cost_panel (GComponent): 货币消耗
  ├─ icon (GImage): 货币图标
  └─ txt_cost (GTextField): "561万/2.00万"

- txt_success_rate (GTextField): "成功率:100%"
- btn_compound (GButton): 合成按钮
- btn_close (GButton): 关闭按钮
```

#### 1.3 创建PC端界面
- 复制A_Compound包
- 重命名为 `A_Compound_PC`
- 调整布局和图标路径(PC端特有)

#### 1.4 导出
- 发布设置: Lua格式
- 导出到: `client/rjengineb_Data/StreamingAssets/Bundles/dev/`

### 步骤2: 配置合成数据

编辑 `Mirserver/Mir200/Envir/data/cfgcsv/Compound.xls` (Excel 97-03格式):

```
ID  一级分组    二级分组    目标道具ID  目标数量  消耗道具1  数量1  货币类型  货币数量  成功率
1   材料合成    残卷        10001       1        10001      5      0        0        100
2   材料合成    残卷×10     10001       10       10001      50     1        20000    100
3   特殊合成    金还丹      20001       1        30001      100    1        50000    80
```

转换为Lua格式:
- 运行Excel转Lua工具(如果有)
- 或手动编辑 `game_config/cfgcsv/Compound.lua`

### 步骤3: 集成到现有系统

#### 3.1 服务端集成
在 `QFunction-0.lua` 中添加(如果edit失败,手动添加):
```lua
-- 在文件顶部require部分添加
require("Envir/QuestDiary/game/compoundMain.lua")
```

#### 3.2 网络消息注册
在 `Message.lua` 或网络处理文件中添加:
```lua
local CompoundMsg = require("Envir/QuestDiary/net/CompoundMsg")

-- 在网络消息分发器中添加
function HandleNetMessage(actor, msgId, data)
    if CompoundMsg.OnNetMsg_Compound(actor, msgId, data) then
        return
    end
    -- 其他消息处理...
end
```

#### 3.3 NPC命令
在NPC脚本中添加命令:
```lua
-- NPC对话脚本中添加
elseif select(1) == "打开合成" then
    compoundMain.OpenCompoundUI(actor)
```

或使用GM命令:
```lua
-- 在gmbox.lua中添加
["合成"] = function(actor)
    compoundMain.OpenCompoundUI(actor)
end
```

### 步骤4: 测试

#### 4.1 启动测试
1. 启动服务端
2. 启动客户端
3. 登录游戏

#### 4.2 打开合成界面
方法1: NPC对话
```
找NPC → 选择"打开合成"
```

方法2: GM命令
```
@合成
```

方法3: 代码调用
```lua
-- 在客户端控制台
FGUI:Open("A_Compound", "compoundMain")
```

#### 4.3 测试流程
1. ✅ 检查一级分组是否正确显示
2. ✅ 切换一级分组,检查二级分组是否更新
3. ✅ 选择二级分组,检查材料消耗区域
4. ✅ 检查材料数量显示(需要/拥有)
5. ✅ 检查货币消耗显示
6. ✅ 检查成功率显示
7. ✅ 点击合成按钮,检查是否成功

### 步骤5: 调试

#### 常见问题排查

**问题1: 界面不显示**
```
检查:
1. FairyGUI资源是否正确导出
2. 路径是否正确: A_Compound/compoundMain
3. 客户端日志是否有错误
```

**问题2: 材料列表为空**
```
检查:
1. Compound.lua配置是否正确加载
2. 当前选中的compoundID是否有效
3. 成本物品配置是否有误
```

**问题3: 合成按钮点击无反应**
```
检查:
1. btn_compound是否正确绑定事件
2. 网络消息ID是否匹配(200001)
3. 服务端是否收到请求
```

**问题4: 材料数量显示异常**
```
检查:
1. 背包系统数据是否正确
2. _GetItemCount函数实现
3. 道具ID是否匹配
```

## 📋 完整示例

### 示例配置: 残卷合成
```lua
{
    ID = 1,
    Group1 = "材料合成",
    Group2 = "二级残卷",
    TargetItemID = 10001,      -- 二级残卷
    TargetItemCount = 1,       -- 获得1个
    CostItem1 = 10001,         -- 消耗二级残卷
    CostItemCount1 = 5,        -- 需要5个
    CostItem2 = 0,             -- 无第二种材料
    CostItemCount2 = 0,
    CostItem3 = 0,
    CostItemCount3 = 0,
    CostCurrencyType = 0,      -- 无需货币
    CostCurrencyCount = 0,
    SuccessRate = 100,         -- 100%成功
    Desc = "5个二级残卷合成1个"
}
```

### 示例配置: 金还丹(复杂合成)
```lua
{
    ID = 9,
    Group1 = "特殊合成",
    Group2 = "金还丹",
    TargetItemID = 20001,      -- 金还丹
    TargetItemCount = 1,
    CostItem1 = 30001,         -- 材料1: 灵石
    CostItemCount1 = 100,
    CostItem2 = 30002,         -- 材料2: 仙玉
    CostItemCount2 = 50,
    CostItem3 = 30003,         -- 材料3: 神晶
    CostItemCount3 = 10,
    CostCurrencyType = 1,      -- 货币类型: 金币
    CostCurrencyCount = 50000, -- 需要5万金币
    SuccessRate = 80,          -- 80%成功率
    Desc = "金还丹合成(3种材料+金币)"
}
```
