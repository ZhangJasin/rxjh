## 🎯 核心功能

### 1. 三级菜单系统
```
一级菜单 (主分类)
  ├─ 材料合成
  └─ 特殊合成

二级菜单 (子分类)
  ├─ 残卷
  ├─ 残卷×10
  ├─ 金还丹
  └─ ...

右侧面板
  ├─ 材料消耗区 (动态扩展1-3种材料)
  ├─ 合成获得区 (目标道具)
  ├─ 货币消耗区 (金币/银币/钻石)
  └─ 合成按钮
```

### 2. 动态材料系统
- 支持1-3种不同材料
- 根据配置自动显示材料数量
- 实时显示玩家拥有数量
- 颜色提示(绿色=足够,红色=不足)

### 3. 货币系统
- 支持3种货币类型
  - 1 = 金币
  - 2 = 银币
  - 3 = 钻石
- 自动检测货币是否足够

### 4. 成功率机制
- 每个合成独立配置成功率(1-100%)
- 合成失败不返还材料
- 成功率可配置

## 🐛 已知注意事项

### 配置注意事项
1. **道具ID必须存在**: 确保配置中的道具ID在Item.lua中有定义
2. **货币类型**: 严格按照 1=金币, 2=银币, 3=钻石 配置
3. **成功率范围**: 建议1-100,不要超过范围
4. **材料数量**: 最多支持3种材料,不需要的填0

### UI注意事项
1. **命名一致**: FairyGUI组件名称必须与Lua代码完全一致
2. **容器组件**: cost_items_container必须是GComponent类型
3. **列表渲染**: 确保列表的defaultItem正确设置

### 服务端注意事项
1. **背包空间**: 合成前检查玩家背包是否有足够空间
2. **并发处理**: 同一时间只能进行一次合成
3. **日志记录**: 所有合成操作都有日志记录
