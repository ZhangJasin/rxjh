# 合成系统 (Compound System)

## 系统概述
合成系统是一个三级菜单结构的道具合成模块,支持材料合成和特殊合成两种类型,允许玩家通过消耗材料和货币来合成目标道具。

## 功能特性

### 1. 三级菜单结构
- **一级菜单**: 主分类 (材料合成 / 特殊合成)
- **二级菜单**: 子分类 (残卷 / 金还丹等)
- **右侧面板**: 材料消耗和合成获得区域

### 2. 动态材料消耗
- 支持1-3种材料消耗
- 自动根据配置动态生成材料显示区域
- 显示玩家需要数量和拥有数量
- 颜色提示: 绿色=足够, 红色=不足

### 3. 货币消耗
- 支持金币、银币、钻石三种货币
- 在材料消耗下方显示
- 显示当前拥有和需要数量

### 4. 合成成功率
- 每个合成配置都有独立的成功率
- 合成失败不会返还材料
- 成功率范围: 1%-100%

### 5. 双平台支持
- 移动端 (A_Compound)
- PC端 (A_Compound_PC)
- 共享数据层逻辑

## 使用方法

### 打开合成界面
```lua
-- 客户端调用
FGUI:Open("A_Compound", "compoundMain")  -- 移动端
FGUI:Open("A_Compound_PC", "compoundMain_PC")  -- PC端

-- 或通过NPC命令
@compound  -- 打开合成界面
```

### 服务端API

#### 打开合成界面
```lua
compoundMain.OpenCompoundUI(actor)
```

#### 执行合成
```lua
compoundMain.StartCompound(actor, compoundID)
```

#### 获取分组列表
```lua
-- 获取所有一级分组
local group1List = compoundMain.GetGroup1List()

-- 获取指定一级分组下的二级分组
local group2List = compoundMain.GetGroup2List("材料合成")

-- 获取合成配置列表
local compoundList = compoundMain.GetCompoundList(actor, "材料合成")
```

## 网络消息

### 消息ID定义
- `200000`: 打开合成界面
- `200001`: 合成请求

### 消息格式

#### 打开合成界面
```lua
-- 客户端 → 服务端
Message:Send(200000, {})

-- 服务端 → 客户端 (返回分组数据)
Message.sendmsgEx(actor, "compoundMain", "initCompoundData", {
    group1List = {"材料合成", "特殊合成"},
    group2Map = {
        ["材料合成"] = {"残卷", "二级残卷×10", ...},
        ["特殊合成"] = {"金还丹", "银还丹", ...}
    }
})
```

#### 合成请求
```lua
-- 客户端 → 服务端
Message:Send(200001, {compoundID = 1})

-- 服务端处理结果通过消息返回
```

## 扩展开发

### 添加新的合成配置
1. 在Excel表中添加新行
2. 转换为Lua格式
3. 重启服务端加载配置

### 添加新的货币类型
1. 在`compoundMainData.lua`的`GetCurrencyCost`中添加新类型
2. 在`compoundMain.lua`的`RefreshCurrencyCost`中添加图标映射
3. 在服务端`compoundMain.lua`的`GetPlayerCurrency`中添加获取逻辑

### 添加合成特效
在服务端`ExecuteCompound`函数中添加特效播放:
```lua
if success then
    -- 播放合成成功特效
    playeffect(actor, "compound_success_effect")
end
```

## 注意事项

1. **配置表同步**: 客户端和服务端的配置表必须保持一致
2. **道具ID**: 确保配置中的道具ID在Item.lua中存在
3. **货币类型**: 
   - 0 = 无货币消耗
   - 1 = 金币
   - 2 = 银币
   - 3 = 钻石
4. **成功率**: 建议范围1-100,100表示必成功
5. **材料数量**: 支持最多3种不同材料
6. **FairyGUI资源**: 需要先创建FairyGUI包并导出资源

## 常见问题

### Q: 合成后没有获得道具?
A: 检查目标道具ID是否正确,以及是否有背包空间

### Q: 材料扣除但合成失败?
A: 这是正常机制,失败不返还材料。如需返还机制,需修改`ExecuteCompound`函数

### Q: 如何添加批量合成?
A: 在配置表中创建多个配置项,如"二级残卷"和"二级残卷×10"

### Q: UI不显示材料列表?
A: 检查FairyGUI中`cost_items_container`容器是否存在,以及CostItem组件是否正确创建
